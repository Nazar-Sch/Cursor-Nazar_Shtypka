Git, GitHub

  • git config --global user.name "Your Name" - вказати ім'я
  • git config --global user.email blabal@bla - вказати email
  • git init - ініцілізувати/створити новий пустий репозиторій
  • git status - показує інформацію про файли, зміни і створення коміттів
  • git add - добавляє файл в репозиторій
  • git commit - бере всі дані, добавлені в індекс за допомогою git add, зберігає їх зліпок; записує зміни в репозиторій
  • git log - історія коммітів в даній branch/вітці
  • git checkout - переключення між двома вітками, віновення старої версії
  • git branch - вітки гіта, створити нову вітку
  • git push - відправляє всі зміни на віддалений репозиторій
  • git pull - отримати дані з віддаленого репозиторію і об'єднати з локальною
  • git merge - вносить зміни з віддаленого репозиторію на локальний, зливає одну або декілька віток в поточну
  • git clone - завантажує репозиторій разом з усією його історією
  • .gitignore - використовується для вказівки в ньому файлів чи папок, які потрібно приховати від системи контролю версій, допомагає застосувати один комміт з одної вітки до дерева іншої
  • git revert - створює новий комміт та скасовує попередні
  • git reset відміна всіх коммітів після вказаного комміта, але залишає всі зміни в робочій директорії
  • git rebase - один з двох способів об'єднати зміни, зроблені в одній гілці, з іншою гілкою
  • git cherry-pick - використовується для того, щоб взяти зміни, внесені кимось, і спробувати застосувати їх заново у виді нового комміта
  • git stash - збереження та відновлення незавершених змін
